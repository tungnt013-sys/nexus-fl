Architecture

How Nexus works

A thin intelligence layer on top of standard Flower FL — no changes to your model or training loop.

01 — Each round

Server calls the agent

After local training completes, server_app.py passes the available client list, per-client accuracy history, and failure counts to the Nexus agent via the evaluate_fn callback.

02 — Agent decides

Claude reasons over clients

The agent (Claude API or heuristic fallback) returns a structured JSON decision: which clients to select, whether to stop early, and a natural-language reasoning string explaining the choice.

03 — Logged & visualized

Every decision is auditable

Decisions are appended to agent_log.json. The Streamlit dashboard reads this file every 3 seconds and renders accuracy curves, client selection history, and agent reasoning in real time.

Agent

What the agent sees

Each round, the agent receives structured client metadata and returns an actionable JSON decision.

Context in → Decision out

The agent prompt includes: round number, available client IDs, which classes each client holds (for non-IID setups), per-client accuracy contributions from previous rounds, and cumulative failure counts.

It returns a selection list, an early-stopping flag, a strategy label, and a plain-English reasoning string — all logged for auditability.

  • ▸ Claude API for language-model reasoning
  • ▸ Heuristic fallback — works with no API key
  • ▸ Structured JSON output, validated each round
  • ▸ Full decision trail in agent_log.json
  • ▸ Live Streamlit dashboard — no setup needed

Setup

Get running in 4 steps

Nexus extends the Flower PyTorch quickstart. If you've run Flower before, this will feel familiar.

1

Clone and install

Clone the repo and install dependencies. Python 3.10–3.11 recommended.

git clone https://github.com/YOUR_USERNAME/nexus-fl.git cd nexus-fl pip install flwr[simulation] torch torchvision anthropic streamlit
2

Set your API key (optional)

Without a key, Nexus uses a diversity-weighted heuristic. The system still runs — you just won't get LLM reasoning in the logs.

export ANTHROPIC_API_KEY=your_key_here
3

Run the FL system

Start the Flower simulation in one terminal.

flwr run . --stream
4

Open the dashboard

In a second terminal, launch Streamlit. It auto-refreshes every 3 seconds.

streamlit run dashboard.py # → open http://localhost:8501
💡 Known gotchas: grid.get_node_ids() returns a set — use sorted() before indexing. evaluate_fn belongs in strategy.start(), not in FedAvg(). See the README for the full list.

Random final accuracy15.05%
Agent final accuracy10.48%
Agent failures vs random21% fewer

The diversity heuristic backfired.

Selecting clients with rare classes sounds smart — but those clients often hold too few samples to produce reliable gradient updates. The heuristic optimized for class coverage without accounting for per-class data volume. Full LLM mode (Claude API reasoning) remains untested at this scale and is the clear next step.
